# World Clean Biz Blog Image Update Plan V1

Created: 2026-06-10

Purpose:

```text
Plan how to update blog images under Visual System V1.1 without damaging archive, chronicle, translation, buying-guide, or document-reference articles.
```

Important rule:

```text
This is a planning document only.
Do not replace public website images or edit article MDX image references until the user explicitly approves a specific article batch.
```

## Visual System

Use:

```text
A Brand Analysis
B Market Analysis
C Founder Profile
D Industry Opinion
E Data Visual
```

## Core Principle

Not every old image should be replaced.

Replace only when a new image improves:

```text
identification
judgment
evidence
reading completion
```

Do not replace images simply to make the site visually uniform.

## Do Not Touch By Default

These article types should generally keep existing body images.

### 1. Chronicle / History / Archive Articles

Reason:

```text
They rely on timeline, historical context, old product photos, screenshots, or accumulated image sequences.
Changing them can destroy the archive feeling.
```

Examples:

```text
hundred-years-of-cleaning-appliance-history
delonghi-group-company-history-and-appliance-brands
delonghi-small-appliance-brand-history
vacuum-industry-2018-review-2019-outlook
first-major-industry-shift-in-2019
midea-cleaning-appliance-past
hundred-years / history / company-history / past / review-outlook style articles
```

Default action:

```text
Do not touch cover or body images unless there is a clear broken image, wrong identity, or user-approved refresh.
```

### 2. Faithful Translation / Report Reading / Annual Report Excerpt Articles

Reason:

```text
Their screenshots and document images are the evidence object itself.
Replacing them with designed graphics can weaken the source-document reading experience.
```

Examples:

```text
irobot-2018-annual-report-faithful-translation
hamilton-beach-2018-annual-report-faithful-translation
ecovacs-2018-annual-report-signals
ecovacs-mid-year-report-reading
groupe-seb-2018-annual-report-signals
tti-2019-interim-report
roborock-ipo-prospectus-signals
roborock-2019-annual-report-breakout-year
nilfisk-2018-annual-report-commercial-cleaning
```

Default action:

```text
Preserve body screenshots / document evidence.
Only consider cover refresh if the current cover is generic or low quality, and only after approval.
```

### 3. Buying Guides / Product Guides

Reason:

```text
Readers need product photos, diagrams, and examples more than designed covers.
```

Examples:

```text
best-robot-vacuum-buying-guide-2022
vacuum-cleaner-buying-guide-2022
hard-floor-washer-buying-guide-2022
complete-hard-floor-washer-buying-guide
complete-vacuum-cleaner-buying-guide-20221006
strongest-robot-vacuum-buying-guide-20221007
```

Default action:

```text
Keep body images.
Only fix incorrect, broken, duplicate, or obviously AI/generic images.
```

### 4. Event / Exhibition Photo Sets

Reason:

```text
Event articles should preserve real event texture and source value.
```

Examples:

```text
awe-china-cleaning-appliance-signals
chinese-cleaning-brands-at-ces-2026
chinese-cleaning-robot-brands-at-ces
cleaning-appliance-companies-at-awe
ifa-2019-vacuum-cleaner-new-products-by-major-brands
vacuum-companies-at-2018-autumn-canton-fair
ces-2026-backyard-robot-war
```

Default action:

```text
Keep real event photos.
Only update cover if it is not representative or if a better real event photo exists.
```

### 5. Patent / Legal Detail Articles

Reason:

```text
Patent drawings and legal screenshots often carry evidence value.
```

Examples:

```text
bissell-vs-tineco-patent-details
bissell-itc-337-investigation-against-tineco
lawsuit-that-shaped-the-handheld-vacuum-industry
lawsuit-that-changed-robot-vacuum-industry-follow-up
two-sides-of-dyson
```

Default action:

```text
Preserve document/patent evidence images.
Only consider adding E Data Visual if the article contains a clear damages/risk/debt number.
```

## Replacement Levels

Use four levels, not one blanket replacement process.

### Level 0: Preserve

```text
No image changes.
```

Applies to:

```text
chronicle
history archive
faithful translation
annual report excerpt
buying guide
document/patent evidence
real event photo sets
```

### Level 1: Add One Data Visual

```text
Keep existing images.
Add or replace one weak body image with an E Data Visual.
```

Best for:

```text
SharkNinja growth
iRobot crisis
Aiper alliance
Maytronics revenue reset
Dyson product comparison
```

### Level 2: Replace Weak Body Images

```text
Replace duplicate, generic, AI-style, or low-information body images with real product / evidence / data visuals.
```

Best for:

```text
Maytronics
Aiper
Dyson
Roborock after data confirmation
```

### Level 3: Cover Refresh

```text
Generate a new cover under A/B/C/D/E.
```

Only after:

```text
source assets are reviewed
article identity is confirmed
user approves the article
```

## Approval Workflow

For each article:

1. Classify the article:

```text
preserve
add-data-visual
replace-body-images
refresh-cover
```

2. Generate candidate image in:

```text
assets/source-library/data-visuals/
assets/source-library/brands/
```

3. Review image visually.

4. User approves.

5. Only then copy final image to:

```text
public/images/insights/
```

6. Only then update MDX references.
